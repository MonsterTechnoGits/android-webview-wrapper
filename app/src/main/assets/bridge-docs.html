⚙️ How it works

The native app injects a Java object named AndroidBridge into every page's window via WebView.addJavascriptInterface(). Once the page has loaded, every method below is callable directly as AndroidBridge.methodName(...) from any JavaScript on the page — no script tag or SDK import required.

Always guard calls with a check, since AndroidBridge is undefined when this page is opened in a normal desktop/mobile browser instead of the wrapper app:
if (typeof AndroidBridge !== 'undefined') { AndroidBridge.showToast('Hello!'); }

Methods come in three shapes: sync returns a value immediately (string/boolean) — async takes a JS callback expression as a string, invoked later by native code — fire-and-forget returns nothing and just performs an action.

Source: app/src/main/java/com/monstertechno/webview/bridge/JavaScriptBridge.java. Registered in WebViewManager.java as "AndroidBridge".

📱 Device / App Info ↑ top

sync AndroidBridge.getDeviceInfo() -> String (JSON)

Returns device platform, OS version, SDK level, manufacturer, model, and brand as a JSON string.

Returns: {"platform":"Android","version":"14","sdk":34,"manufacturer":"Google","model":"Pixel 8","brand":"google"}

sync AndroidBridge.getAppInfo() -> String (JSON)

Returns this app's package name, versionName, and versionCode.

Returns: {"packageName":"...","versionName":"1.0","versionCode":1}

sync AndroidBridge.isFeatureSupported(feature: String) -> boolean

Params: feature — one of "fingerprint", "camera", "microphone", "location", "vibrator"

Checks whether the device hardware/feature is available before you try to use it.

📣 Toast / Vibrate ↑ top

fire-and-forget AndroidBridge.showToast(message: String): void

Shows a native Android Toast. Always runs on the UI thread internally — safe to call from any JS context.

fire-and-forget AndroidBridge.vibrate(milliseconds: int): void

Triggers a one-shot vibration. No-op on devices without a vibrator (already checked internally).

🔔 Notifications ↑ top

fire-and-forget AndroidBridge.showNotification(title: String, message: String, iconUrl: String): void

Params: title, message — notification text. iconUrl — remote image URL, pass an empty string for the default app icon.

Posts a system notification through the app's notification channel. Requires the POST_NOTIFICATIONS runtime permission on Android 13+ (already requested by PermissionManager).

🔐 Biometric Authentication ↑ top

async AndroidBridge.authenticateFingerprint(callback: String): void

Params: callback — a JS function expression as a string, e.g. "function(success, message) { ... }". Native code invokes it by name after the system biometric prompt resolves.

The callback string is evaluated as raw JavaScript by the native layer (webView.evaluateJavascript()). Only pass a literal function expression — never interpolate untrusted/remote text into this string, since it executes with full page privileges.
AndroidBridge.authenticateFingerprint( 'function(success, message) {' + ' AndroidBridge.showToast(success ? "Auth OK" : "Failed: " + message);' + '}' );

Check availability first with isFeatureSupported('fingerprint') before showing a biometric-only UI path.

📁 Files ↑ top

fire-and-forget AndroidBridge.downloadFile(url: String, filename: String): void

Downloads a remote file in the background via WorkManager and saves it to the device's Downloads folder. Survives app backgrounding/kill.

fire-and-forget AndroidBridge.shareFile(filePath: String): void

Params: filePath — an absolute path on local storage (typically one previously downloaded via downloadFile).

Opens the system share sheet for the given local file via a FileProvider URI.

🌐 Share / External Apps ↑ top

fire-and-forget AndroidBridge.shareText(text: String): void

Opens the system share sheet (ACTION_SEND) with plain text.

fire-and-forget AndroidBridge.openUrl(url: String): void

Opens any URL with ACTION_VIEW, letting Android route it to the best-matching app (browser, maps, mail client, etc). Unlike normal links inside this WebView, this always leaves the app.

fire-and-forget AndroidBridge.openExternalApp(packageName: String): void

Params: packageName — e.g. "com.android.settings". No-op (silently does nothing) if the package isn't installed.

⬅️ Back-press Control ↑ top

fire-and-forget AndroidBridge.setBackHandled(handled: boolean): void

Lets the web page claim the Android back button/gesture while it has an open modal, drawer, or bottom sheet — so back closes your overlay instead of navigating WebView history or exiting the app.

  1. Call AndroidBridge.setBackHandled(true) when your overlay opens.
  2. Define window.__onAndroidBack = function() { /* close overlay */ } — native code calls this when back is pressed while handled.
  3. Call AndroidBridge.setBackHandled(false) when your overlay closes normally.
let modalOpen = false; function openModal() { modalOpen = true; AndroidBridge.setBackHandled(true); window.__onAndroidBack = closeModal; // ...show your modal } function closeModal() { modalOpen = false; AndroidBridge.setBackHandled(false); // ...hide your modal }

Works with any framework — there are no required CSS classes or DOM markup conventions, just the two bridge calls plus your own open/close logic.

⚠️ Error Handling ↑ top

Bridge methods don't throw structured errors back to JS — sync methods that hit an internal exception return a JSON object with an "error" key (e.g. getAppInfo()) instead of throwing. Fire-and-forget methods fail silently on the native side if a precondition isn't met (e.g. openExternalApp with a missing package, or vibrate on a device with no vibrator).

Always wrap bridge calls in try/catch when parsing JSON results, and check typeof AndroidBridge !== 'undefined' before calling anything, since this page (and your real site) may also be opened in a regular browser during development.

function safeCall(fn) { if (typeof AndroidBridge === 'undefined') { console.warn('AndroidBridge not available - not running inside the wrapper app'); return null; } try { return fn(); } catch (e) { console.error('Bridge call failed:', e); return null; } }
